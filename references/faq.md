# 常見問題 · FAQ

> 給 ARTOGO Design Master skill 在同事問「怎麼用」「我有哪些觸發詞」「忘記怎麼做了」時參考。
> 用平易語氣回答，直接給範例，不要講原理。

---

## Q1 · 我要怎麼開始做設計？

直接說你要做什麼就好。範例：

| 你想做什麼 | 直接這樣說 |
|----------|-----------|
| IG 貼文圖 | 「幫我做一張 IG 封面，主題是下週活動」 |
| 提案 PPT | 「做一份提案 PPT 封面，主題是 Q2 規劃」 |
| 公司海報 | 「做一張 A4 直式海報，講 5/15 開幕活動」 |
| 動畫 | 「做一個 10 秒動畫，介紹我們新品」 |
| 多圖科普 | 「把這段內容做成 6 張多圖科普」 |
| 信息圖 | 「把這份資料做成一張信息圖」 |

我會自動套用 ARTOGO 設計規範，不用特別講。

## Q3 · 我可以指定設計風格嗎？

可以，但**不能違反 ARTOGO 核心規範**（深色 / 金棕 / Roboto + Noto Sans TC）。

可以調整的：
- ✅ 版面布局（單欄 / 多欄 / 圖文並列）
- ✅ 字級節奏（標題大 / 標題小 / 全篇等大）
- ✅ 配圖比例（大圖 / 小圖 / 無圖）
- ✅ 氛圍（嚴肅 / 活潑 / 安靜）

**不能改**：
- ❌ 主色（必須金棕 #ba9972）
- ❌ 底色（必須深色 #191919）
- ❌ 字型（必須 Roboto + Noto Sans TC）

如果你要做完全不同風格的設計（例如完全淺色、彩色多色、不同字型），請改用 huashu-design 直接做，告訴它「不套用 ARTOGO 規範」。

---

## Q4 · 我可以一次做很多張嗎？

可以。範例：

> 「幫我做 6 張 IG 多圖，分別講 [六個主題]，套 ARTOGO 風格」

我會：
1. 先做 1 張 showcase 給你看，定方向
2. 你點頭後，再批量推剩下 5 張
3. 最後做一張 index 拼接頁，方便預覽

---

## Q5 · 做完的檔案在哪？

預設放在你的 `Desktop` 下，會建一個資料夾（資料夾名跟你的需求有關），裡面有：
- 每張獨立的 HTML 檔案（雙擊就能在瀏覽器開）
- 一個 `index.html`（拼接預覽）
- `brand-spec.md` 紀錄這次用了什麼規範

如果要 PNG / PDF，跟我說「**幫我截成 PNG**」「**幫我輸出 PDF**」，我會處理。

## Q7 · 出問題怎麼辦？

跟我說：
- 「**ARTOGO 跑不起來**」
- 「**設計工具壞了**」
- 「**找不到觸發**」

我會帶你疑難排解。如果是更嚴重的問題（例如整個 skill 不見了），跟我說：

> 「**重新設定 ARTOGO 設計工具**」

我會重新跑一次完整安裝流程。

---

## Q8 · 公司同事也想用怎麼辦？

請他打開 Claude Code，貼這行到對話框（請維護者把 `YOUR_GITHUB` 換成實際帳號）：

```
請執行：npx skills add YOUR_GITHUB/artogo-design-master -g --agent claude-code -y
```

跑完後在新對話打「我要開始用 ARTOGO 設計工具」，就會跑到 Step 1 設定流程。

---

## Q9 · 升級規範了怎麼辦？

當 ARTOGO Design System 有更新（新色、新字、新規範），維護者會 push 到 GitHub。

同事的電腦要拿到最新版，跟我說：「**升級 ARTOGO 設計工具**」，我會幫你跑：

```bash
npx skills update artogo-design-master
```
